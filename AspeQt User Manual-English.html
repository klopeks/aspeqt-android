AspeQt User Manual

This manual describes the current AspeQt build included with this project. It focuses on the features that are available today, especially on Android.

What AspeQt Does

AspeQt is an Atari 8-bit serial peripheral emulator built with Qt. It can emulate disk drives, folder-based images, cassette playback and printer output through a SIO2PC-compatible connection.

  • 6 disk stations
  • ATR, XFD, ATX and PRO disk image support
  • Folder images
  • Direct booting of Atari executable files such as XEX, COM and EXE
  • Cassette playback from CAS files
  • Printer emulation with text preview
  • Session save and restore

Main Screen

Each disk row contains a station number and action buttons for mounting, saving, editing and ejecting media. On Android, the interface is optimized for portrait orientation. When a disk or folder image is mounted, its file name is shown below the row.

The message area below the disk rows shows runtime events such as mounted images, printer state and emulator start/stop notifications.

Disk Images

You can mount a disk image in several ways:

  • Use the disk button in a station row
  • Use the main menu command for mounting a disk image
  • Open a recent image from the recent list
  • Drag and drop an image onto a disk row

After mounting, the Atari sees the selected image as a drive. You can then save changes, enable or disable write protection, open the image explorer or eject the image.

Folder Images

A folder image exposes a host folder as Atari media. In the interface it is shown with a different label color so it can be distinguished from a regular disk image.

Booting Atari Executables

AspeQt can boot executable files directly without creating a disk image first. You can open an executable from the menu or by dragging it into the window.

Before booting an executable, AspeQt now prepares the emulator for a clean start. On Android this means the emulator is stopped, mounted disks are removed and the emulator is started again before the file is loaded.

After the file is loaded, the executable dialog stays open and offers:

  • Retry to boot the same file again
  • Open to pick another executable without returning to the main screen
  • Cancel to close the dialog

A high speed executable loader is also available in the emulation options. Compatibility still depends on the program being loaded, so some executables may require the normal loader.

Cassette Playback

CAS files can be played back through the cassette command. AspeQt can use the baud rate stored in the image or a custom value configured in the options.

Printer Emulation

AspeQt emulates the Atari text printer device P1:. You can start or stop printer emulation from the main window. Printer output can be previewed, saved or printed.

Options and Language

The options dialog stores hardware, speed and language settings. If the language is set to auto, AspeQt uses the system locale. If you choose a specific language, the interface follows that preference instead.

The user manual window also follows the selected application language when a translated manual is available.

Sessions

Sessions let you save the current disk setup and selected configuration so that the same environment can be restored later.

Notes

  • Some advanced features depend on the connected SIO2PC hardware and the selected serial backend.
  • High speed modes depend on cable quality, handshake method and Atari-side software support.
  • Direct executable booting is convenient, but not every Atari program is compatible with it.
